진행사항:
1. api 개발 계속 진행중.
2. 사용자 인증 방식 구현 완료 -> problem 을 내면 encrypt 해서 Answer 을 제출하는 방식으로 해결함.
3. 파일 쪼개서 분할하려면 그룹을 만들어야 하는데 이 그룹 기능을 만들려면 참여하는 사용자들에게 동의를 구해야함. -> 즉 이걸 동의를 어떻게 받을래? HTTP 로 한명씩 동의할때까지 기다릴까?
    아니면 서버에서 TCP 연결을 중개해줄까? 라고 생각했는데 후자가 너무 불필요해.
    (새로운 방법 생각남... 가장 좋은 방법인듯... -> 그룹에 끼고 싶은 유저들은 flag 를 사용해서 그룹에 들어가고 싶음을 표시함 -> 누군가 그룹을 만들고 싶다는 요청을
       서버에게 보내면 서버는 그룹에 포함되어있지 않은 유저들 중에서 flag를 true 로 세팅한 유저들에 대해서 그룹을 만들어주면 됨. 물론 그룹에 참여했다는 flag 와 참여의사 flag 는 서로
       반전 관계여야함. 그룹에 포함되는 순간 다른 그룹에는 포함될 수 없음.)
